# openFrameworksを使ったアプリケーションの基本構造

## setup / update / draw / exit

* setup
  * アプリケーションの初期化
    * 設定ファイル等の必要なデータの読み込み
    * 事前に分かっているデータの計算
    * カメラの初期化
    * ネットワーク通信の初期化
    * etc, etc...
* update
  * アプリケーションの状態やデータの更新
    * カメラの更新
    * OSCの受信処理
    * FBOへの描画
    * etc, etc...
* draw
  * 描画処理
* exit
  * 終了処理（アプリケーション自体が終了するのでそこまで気にしなくても良い）
    * 必要なデータの永続化
    * キャッシュの削除
    * スレッドの終了
    * etc, etc...
